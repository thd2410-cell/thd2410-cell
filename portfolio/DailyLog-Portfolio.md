# DailyLog 포트폴리오

## 프로젝트 한 줄 소개
AI 체크인으로 기록의 진입장벽을 낮추고, 기록 데이터를 개인화 추천으로 연결해 행동 변화를 돕는 서비스입니다.

## 프로젝트 개요
DailyLog는 사용자의 하루를 AI 체크인 방식으로 기록하고, 기록된 감정/행동 데이터를 바탕으로 다음 행동을 추천하는 서비스입니다.  
기존 기록 서비스의 한계(긴 글 작성 피로, 기록-행동 단절)를 해결하기 위해 **질문 기반 입력 → 자동 생성 → 인사이트 → 추천 → 피드백**으로 이어지는 순환 구조를 설계했습니다.

---

## 문제 정의
### 1) 기록의 피로도
- 많은 사용자가 기록 필요성에는 공감하지만, 매일 긴 문장 작성 과정에서 이탈
- 기록 시작까지의 진입장벽이 높아 습관 형성이 어려움

### 2) 기록 데이터의 가치 전환 부족
- 기록이 쌓여도 분석/실행으로 연결되지 않으면 사용자 체감 가치가 낮음
- 감정/행동 데이터를 실제 개선 행동으로 연결하는 장치 필요

---

## 해결 방식
DailyLog는 AI 체크인 흐름 중심으로 다음 구조를 구현했습니다.

1. 사용자가 질문(사건, 감정, 원인, 하이라이트, 어려움, 내일 행동)에 답변
2. 시스템이 답변을 바탕으로 일기와 감정 인사이트를 자동 생성
3. 생성된 데이터를 개인화 추천으로 연결
4. 사용자 피드백(평점/수행 여부)을 저장해 추천 품질 개선에 반영

즉, **입력 부담 완화**에서 끝나지 않고 **기록-분석-추천-피드백**의 폐쇄 루프를 구축했습니다.

---

## 담당 역할 및 기여

### 1) 백엔드 아키텍처 설계
- Java 21 + Spring Boot 기반 백엔드 구성
- `adapter` / `application` / `domain` 분리의 Hexagonal Architecture 적용
- 웹, 영속성, 외부 AI 연동 계층 간 결합도 축소 및 확장성 확보

### 2) AI 서버 분리 및 연동
- FastAPI 기반 AI 서버를 별도 운영
- 체크인 질문 생성, 일기 생성, 임베딩, 추천 계산 책임 분리
- 백엔드-모델 연산 경계 명확화로 모델 교체/프롬프트 변경 유연성 확보

### 3) 추천 흐름 설계
- 일기 생성 결과에서 감정/행동 태그/축 정보 추출
- 이를 기반으로 추천 후보 구성
- 사용자 평가/수행 로그를 반영하는 피드백 루프 설계

### 4) 비동기 이벤트 파이프라인 구축
- Kafka 기반 `diary-generated` 이벤트 발행
- 소비자에서 후속 추천 후보 확장 작업 수행
- 사용자 응답 지연 없이 추천 카탈로그를 점진적으로 강화

## 기술적 핵심 포인트

### AI 기능의 서비스화
- AI 기능을 단순 호출이 아닌 **분리된 운영 단위**로 설계
- 장애 격리, 응답 지연 완화, 변경 비용 절감을 고려한 구조 채택

### 추천 시스템의 복합 설계
- 룰 기반 판단 + 임베딩 유사도 검색 + 피드백 반영 결합
- PostgreSQL + pgvector 기반 유사 기록/행동 후보 탐색
- 단발 추천이 아닌 장기 성능 개선 구조 확보

### 운영 안정성 강화
- Redis 기반 Rate Limit
- Request ID 로깅
- 메트릭 수집
- Flyway 마이그레이션 관리

AI 호출이 포함된 서비스 특성을 고려해 기능 구현과 운영 가능성을 함께 확보했습니다.
